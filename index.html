Maze Game

This project aimed to showcase my comprehensive understanding of classes, objects, and arrays. The primary objective of the game was to navigate the player (represented by the symbol A) through a maze while avoiding walls (symbolized by #) and reaching the exit. Along the way, the player encountered various obstacles, including vanishing items (symbolized by v) that concealed their position. A console mode, accessible through the tilda key, provided essential information about the player, such as their current position and the number of vanishes until reaching the exit. Furthermore, recent builds of the game introduced additional features like a maze builder, the ability to import mazes, and customizable settings. During the development process, I encountered challenges with the maze import functionality, primarily due to inconsistencies in reading the file caused by differences between the endl and \n characters. To overcome this hurdle, I implemented a solution that selectively reads viable characters, ensuring that the exact format of the stored array became less critical.

Bookstore

The Bookstore program was completed for the course COP 3330 (OOP) at Florida State University. The purpose of this game was to demonstrate my ability to use multifile compilation, pointers, and dynamic memory allocation to create a cohesive store. The program works through a dynamically allocated list of books (objects of class) within a register object. This solves the issue of scalability, since this program may hold as many books as allowed by your computer's memory. This program taught me the importance of abstraction within object-oriented programming, as well as how to build a cohesive operation that scales to the user's needs.
